7.Булева логика

Описания логических операторов приведены в таблице:

Оператор Использование Описание
Логическое И (&&) expr1 && expr2 Возвращает значение expr1, если оно может быть преобразовано в false; иначе возвращает значение expr2. Таким образом, при использовании с величинами типа Boolean оператор && вернет true, если оба операнда могут быть преобразованы в true; иначе оператор && вернет false.
Логическое ИЛИ (||) expr1 || expr2 Возвращает значение expr1, если оно может быть преобразовано в true; иначе возвращает значение expr2. Таким образом, при использовании с величинами типа Boolean оператор || вернет true если хоть один из них равен true; в других случаях вернет false.
Логическое НЕ (!) !expr Возвращает false если значение expr можно привести к true; в противоположном случае возвращает true.
Примеры значений выражений, которые могут быть преобразованы в false:
  • null;
  • NaN;
  • 0;
  • пустая строка ("");
  • undefined.
Примечание: Хоть операторы && и || могут использовать операнды с не булевыми значениями, но они всёравно рассматриваются, как булевы операторы, т.к. их возвращаемые ими значения всегда могут быть сконвертированы в булевы значения.

Короткая схема вычислений

Так как логические операторы выполняются слева направо, они проверяются на "короткие вычисления" по следующим правилам:
  • false && (anything) короткое вычисление дающее false
  • true || (anything) короткое вычисление дающее true
Примечание: Часть выражения (anything) не вычисляется. Если в ней есть вызов функции, то эта функция не будет вызвана.

Помимо десятичных целых чисел JavaScript позволяет записывать числа в восьмиричном формате когда сначала пишется «0», за которым следуют цифры 0-7, а также существует запись в шестнадцатеричных значениях. 2.Новый синтаксис, понятие деструктуризации объекта

Новый синтаксис

Допустим у нас есть объект newPerson и мы хотим получить значения полей этого объекта в отдельные переменные:

const newPerson = {

name: 'Egor',

age: 40,

isProrrammer: true,

languages: ['ru','en'],

greet(){ // Воспользовались новым синтаксисом о котором говорилось ранее

alert('Greet!!!')

}

}

name = newPerson.name

age = newPerson.age

languages= newPerson.languages

Если посмотреть теперь на эти переменные через console.log(name, age, languages), то мы увидим их как отдельные переменные, т.е. они представляют собой ссылки на поля объекта

newPerson:

Egor 40 > (2) [ "ru" , "en" ]
В предыдущем коде, выше ⬆ можно увидеть много дублирующихся имен: name,age..., которые являются переменными, с левой стороны выражения, а с правой стороны выражения они представляют собой свойства объекта.
Чтобы избежать такого дублирования можно воспользоваться деструктуризацией объекта, и тогда запись кода, где мы присваиваем отдельным переменным свойства объекта newPerson будет следующей:
const {name, age, languages } = newPerson // это и есть деструктуризация объекта
Результат вывода переменных командой console.log(name,age,languages) будет таким же, т.е. получение переменными свойств объекта newPerson, записаное тремя строками равнозначно одной строке, записаной в новом синтаксисе.
Новый синтаксис, также позволяет переименовывать ключи объекта, приэтом значения ключей остаются прежними, и делается это следующим образом:

const{name,age:fieldAge,languages,...rest} = newPerson

const newObj = {name, fieldAge, languages,...rest}

Замечание: Переменная rest будет содержать те значения, которые не были присвоены переменным name, fieldAge и languages
